CCP安装前准备

网络拓扑

CCP的整体架构包含两个部分：

  • CCP 控制平面 （CCP Ctrl）

    用于CCP的安装和用户K8s集群管理，包括KCP installer， CCP Ctrl集群

  • CCP 数据平面 （CCP Group）

    是真正由用户使用的K8s集群，由CCP Ctrl集群创建

CCP安装必须满足以下条件：

  • KCP installer, DHCP server和CCP Ctrl集群必须在同一个subnet （在本环境中，使用跳板机Jump box实现）
  • control-plane网络和k8s tenant网络保证至少有/24网络地址
  • control-plane网络和K8s tenant网络之间可以正常通讯
  • control-plane网络和K8s tenant网络可以正常访问Internet

地址/VLAN

CCP 控制平面 （CCP Ctrl）

  • CCP ctrl-plane IP（Installer VM SUBNET CIDR）:

    KCP installer, DHCP server, 跳板机，CCP ctrl-plane cluster都在此网段。在ACI上创建pov-ccp-ctrl tenant用于CCP控制平面的provision

    172.16.60.1/24

  • CIDR for kubernetes pod network (KCP VM parameter)

  • CIDR For Controller Kubernetes Pod network(KCP Installer参数，缺省配置不用修改): 

    192.168.0.0/16

NTP: 

10.66.141.50
10.66.141.51

DNS: 

64.104.123.245

CCP 数据平面 （CCP Group）

  • node Routable CIDR: (配置位置在ACI CNI，用于Group cluster node)

    10.50.0.1/16

  • subnet for Pods(ACI-CNI): 

    10.51.0.1/16

  • subnet for service(ACI-CNI): 

    10.52.0.1/16

  • node vlan start ID: 680 (HX FI trunk allow)

  • node vlan end ID: 700 (HX FI trunk allow)10

ACI环境设置

  1. 设置Tenant/VRF/BD/EGP用于CCP control-plane nodes

    • tenant/epg: pov-ccp-ctrl/ccp-ctrl (consume common/default contract)
    • VRF: common/default
    • BD: subnet 172.16.60.1/24, 关联L3out
    • EPG: KCP installer, DHCP server, 跳板机，CCP ctrl-plane cluster都放在这个EPG中



  2. 配置L3out in common tenant用于访问Internet，下载相关文件

    按照标准方法在common tenant中配置L3out（过程省略）

  3. 在外联路由器上配置返回路由和NAT配置

    ip route 172.16.60.0/24 192.168.254.1

ip nat inside source list NAT interface Ethernet1/2 vrf default overload 
IP access list NAT
30 permit ip 172.16.60.0/24 any

DHCP Server设置

使用ASAv提供DHCP功能，ASAv配置如下：

interface GigabitEthernet0/0
 nameif inside
 security-level 100
 ip address 172.16.60.254 255.255.255.0

dhcpd option 3 ip 172.16.60.1
!
dhcpd address 172.16.60.10-172.16.60.200 inside
dhcpd enable inside
